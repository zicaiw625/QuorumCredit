# Requirements Document

## Introduction

This feature adds a test that verifies `get_loan` returns `None` when called with an address that has no loan record. The test targets the `QuorumCreditContract` in the Rust/Soroban lending protocol and follows the existing test file conventions in `QuorumCredit/src/`.

## Glossary

- **Contract**: The `QuorumCreditContract` Soroban smart contract under test.
- **Client**: The `QuorumCreditContractClient` generated by the Soroban SDK, used to invoke contract methods in tests.
- **Fresh_Address**: An `Address` generated during the test that has never requested a loan.
- **get_loan**: The contract method `get_loan(env, borrower) -> Option<LoanRecord>` that returns the latest loan record for a borrower, or `None` if none exists.

## Requirements

### Requirement 1: get_loan Returns None for Address With No Loan Record

**User Story:** As a protocol developer, I want a test that calls `get_loan` on a fresh address, so that I can confirm the contract returns `None` when no loan record exists for that address.

#### Acceptance Criteria

1. THE Test SHALL create a standard test environment using `Env::default()` with `mock_all_auths()` and a registered `QuorumCreditContract` instance, following the setup pattern used in existing test files.
2. WHEN `get_loan` is called with a `Fresh_Address` that has never requested a loan, THE Client SHALL return `None`.
3. THE Test SHALL assert that the returned value `is_none()`, failing with a descriptive message if the assertion does not hold.
4. THE Test SHALL be placed in a new file `QuorumCredit/src/get_loan_none_test.rs` and declared as a `#[cfg(test)]` module in `QuorumCredit/src/lib.rs`.
